# Design System

## Семантические токены

Все цвета, градиенты, тени — **семантические design tokens**, определённые в
глобальном CSS (`src/styles.css`) и темизированные через shadcn-варианты.

**НИКОГДА** не хардкодить color-утилиты в компонентах:
- ❌ `text-white`, `bg-black`, `bg-[#...]`
- ✅ `text-foreground`, `bg-background`, `bg-primary`

Хардкод обходит темизацию и ломает dark mode.

## Отказ от generic AI-эстетики

**Отвергать:**
- Шрифты по умолчанию: Inter, Poppins
- Фиолетово-индиго градиенты на белом
- Взаимозаменяемые hero/nav/footer layout

**Делать:** один отличительный, целостный визуальный направление для каждого
проекта. Никакие два проекта не должны выглядеть одинаково.

## Шрифты

Загружать через `<link>` в `src/routes/__root.tsx` head, не `@import` в CSS.

## Темы

Светлая и тёмная темы через CSS-переменные:

```css
:root {
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  /* ... */
}

.dark {
  --background: 222.2 84% 4.9%;
  --foreground: 210 40% 98%;
  /* ... */
}
```

## Современные практики (2026)

- OKLCH цвета для перцептуальной равномерности
- Glassmorphism (`backdrop-blur`)
- Кастомные `@utility` в Tailwind v4
- Плавные анимации через Framer Motion
- Mesh-градиенты (radial gradients)
- Адаптивная типографика (clamp())
